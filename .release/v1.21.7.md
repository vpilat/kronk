# Release Notes - v1.21.7

**Release Date:** April 3, 2026

## Overview

This release fixes a critical bug in the Responses API where tool call arguments were not being properly serialized and emitted. It also adds configuration support for Google's Gemma 4 31B model variants with incremental message caching (IMC) enabled, and removes redundant sampling parameters that were inadvertently included in the Cline configuration.

## Bug Fixes

### Responses API Tool Call Arguments (#442)

Fixed a significant bug in the `sdk/kronk/response.go` file where tool call arguments were not being properly emitted in the Responses API streaming output:

- **Changed `Arguments` field type**: Modified `ResponseOutputItem.Arguments` from `string` to `*string`. This ensures function_call items always emit the Arguments field (even when empty) while message items omit it entirely due to `omitempty`.

- **Streaming state consistency**: Rewrote the `complete()` function to rebuild output items from streaming state rather than relying on in-progress values. This ensures that:
  - Tool call IDs match what was already streamed to the client
  - Arguments are properly accumulated and emitted with correct status

- **Empty arguments handling**: When a `function_call` item is first created, it now initializes with an empty string pointer (`Arguments: &emptyArgs`) rather than omitting the field.

- **Double-encoding fix**: Changed how tool call arguments are serialized from `json.Marshal(tc.Function.Arguments)` to `json.Marshal(map[string]any(tc.Function.Arguments))`. This avoids double-encoding caused by the custom `MarshalJSON` on `ToolCallArguments` which was wrapping arguments in an extra JSON string layer for Chat Completions.

## New Features

### Gemma 4 31B Model Support

Added configuration profiles for Google's Gemma 4 31B model variants with incremental message caching enabled:

**gemma-4-31B-it-UD-Q8_K_XL/CLINE:**
- `nseq-max`: 1 (single sequence for IMC)
- `incremental-cache`: true
- Sampling: temperature=1.0, top_k=64, top_p=0.95

**gemma-4-31B-it-UD-Q8_K_XL/KILO:**
- `nseq-max`: 3 (supports multi-user IMC sessions)
- `incremental-cache`: true  
- Sampling: temperature=1.0, top_k=64, top_p=0.95

## Configuration Changes

### Removed Redundant Parameters

Removed `dry_multiplier` and `presence_penalty` parameters from the `gemma-4-26B-A4B-it-UD-Q8_K_XL/CLINE` and `gemma-4-26B-A4B-it-UD-Q8_K_XL/KILO` configurations in:
- `sdk/tools/defaults/default_model_config.yaml`
- `zarf/kms/model_config.yaml`

These parameters were not being applied correctly and have been removed to simplify the configuration.

## Migration Notes

No breaking changes for existing clients, but users of the Responses API should be aware:

1. **Tool call arguments are now always present**: The `Arguments` field in `ResponseOutputItem` is now a pointer, meaning it will be included (possibly as null) rather than omitted entirely for function_call items.

2. **Empty arguments**: When a tool call is created with no arguments, the field will now be present with an empty string value instead of being omitted.

3. **Configuration updates**: If you have custom model configurations for Gemma 4, you may want to review and align them with the updated default configuration that removes redundant DRY and presence penalty parameters.
