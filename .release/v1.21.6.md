# Release Notes - v1.21.6

**Release Date:** April 2, 2026

## Overview

This release adds official support for Google's Gemma 4 model family, including proper handling of its unique tool call format and structural markers. It also includes configuration documentation improvements and a bug fix in the model listing endpoint.

## New Features

### Gemma 4 Model Support (#440)

Full integration for Google's Gemma 4 models with support for their distinctive output formats:

- **Tool Call Parsing**: New `parseGemmaToolCallFormat()` function handles the unique Gemma 4 tool call syntax:
  - Format: `call:get_weather{location:<|"|>New York City, NY<|"|>}`
  - Properly parses nested braces and handles `<|\"|>` escaped quote tokens
  - Supports multiple tool calls in a single response

- **Reasoning Channel Handling**: Added support for Gemma 4's `<|channel|>` and `<channel|>` structural markers:
  - Automatically detects reasoning channels (e.g., `thought`)
  - Streams content appropriately as reasoning mode

- **Tool Call Markers**: Extended token handling for Gemma 4 structural tags:
  - `<|tool_call>` and `<tool_call|>` for tool invocation boundaries
  - `<|tool_response>` and `<tool_response|>` for response markers

## Bug Fixes

### Tool Call Parsing Fix

- Fixed `parseFunctionFormat()` to correctly extract function bodies by calculating the proper body start position when searching for closing `</function>` tags

### Model Listing Endpoint

- Fixed a bug in the `/v1/models` endpoint where tokenizer fingerprints were being accessed on invalid model entries
- Changed from indexed to range-based loop to avoid nil pointer issues

## Documentation Updates

### Model Configuration Guide

Enhanced the default model configuration file (`sdk/tools/defaults/default_model_config.yaml`) with:

- **Cache Session Management**: Added `max-cache-sessions` parameter documentation (default: 1, supports multiple concurrent IMC users)
- **Jinja Template Override**: New `template` option for specifying custom template files
- **IMC Context Auto-Scaling**: Detailed explanation of how Kronk automatically scales internal context windows when IMC is enabled:
  - Formula: `Internal NCtx = context-window × (nseq-max + max-cache-sessions)`
  - Example: 128k context with 1 NSeq and 1 cache session = 256k internal context

- **SPC vs IMC Clarification**: Updated guidance on when to use each caching strategy:
  - SPC: Models with inconsistent templates (GPT-OSS, GLM), simple chat apps
  - IMC: Models with consistent templates (QWEN, Llama), agentic workflows

- **Agent Client Notes**: Added recommendations for Cline and coding agent integration

### Model Selection Guide

Added comprehensive documentation about quantization formats:
- F16 (FP16): Unquantized, maximum quality
- Q8_0: 8-bit near-lossless compression
- K_XL: K-quant smart method with better accuracy at similar sizes

## Migration Notes

No breaking changes. Existing configurations remain compatible. Users wanting to enable Gemma 4 support should:
1. Download a Gemma 4 model from the catalog
2. Use `/IMC` suffix for agentic workflows: `google/gemma-4-x-it-Qx_K_M/IMC`
3. Set `KRONK_CACHE_ID` header for multi-user IMC sessions
