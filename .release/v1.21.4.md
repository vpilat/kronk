# Release Notes - v1.21.4

**Release Date:** March 16, 2026

## Overview

Version 1.21.4 brings significant enhancements to the VRAM calculator, improved template configuration options, and updated dependencies to support the latest llama.cpp versions.

---

## 🚀 New Features

### Template Configuration in Model Config (#424)
- **Author:** Florin Pățan
- Allow model configuration to define the chat template directly
- The `model_config.yaml` can now specify a template model for each model entry
- This provides greater flexibility in template management without catalog changes

**Example configuration:**
```yaml
google/gemma-3-4b-it-Q4_K_M:
  template-model: mistralai/Mixtral-8x7B-Instruct-v0.1-Q5_K_M
```

---

## 🔧 Improvements

### VRAM Calculator Enhancements (#423, #425)
- **Authors:** William Kennedy
- Complete overhaul of the VRAM calculator with improved accuracy and features
- Added support for shorthand URLs in HuggingFace model resolution
- Enhanced model info caching with LRU eviction and 14-day TTL
- Unified VRAM calculator to allow resource overrides
- Forward model config to catalog editor for better integration

**Key improvements:**
- GGUF header caching with atomic file writes
- Hardened HTTP Range request handling
- URL normalization (blob/ → resolve/, hf.co → huggingface.co)
- Better memory estimation for MoE models

### BUI Improvements (#421)
- **Author:** Florin Pateful
- Added parameter tooltips throughout the Browser UI
- Unified labels across all BUI components
- Improved VRAM calculator panel with better resource override options
- Added web_page field exposure in model info API response

### Library Updates
- Updated to latest yzma for compatibility with newest llama.cpp (#422)
- Updated test workflow to use llama.cpp version b8297 (#419)
